三张 Mermaid 图，说清火鸡问题的全部逻辑。

## 图一：火鸡的认知 vs 真实世界

<pre class="mermaid">

graph TB
    subgraph 火鸡看到的世界
    A[每天有食物] --> B[归纳: 明天也有]
    B --> C[结论: 我很安全]
    end

    subgraph 真实的世界
    D[农夫] -->|投喂| A
    D -->|隐藏目标| E[感恩节屠宰]
    end

    C -.-x E

    style C fill:#4caf50,color:#fff
    style E fill:#d32f2f,color:#fff

</pre>

火鸡的模型是自洽的——364 天全部验证。但它不知道系统里有一个变量叫"感恩节"。

火鸡最大的错误不是数据不够，是**不知道系统还有另一个参与者的日程表**。

## 图二：火鸡触发的五个认知偏误

<pre class="mermaid">

graph TD
    A[🔥 致命结论: 我很安全] --> B[简单联想: 农夫=食物]
    A --> C[避免不一致: 说了364天不能改口]
    A --> D[过度乐观: 今天安全=永远安全]
    A --> E[社会认同: 其他火鸡也这么想]
    A --> F[确认偏误: 只记录投喂, 忽略同类消失]

    style A fill:#e65100,color:#fff

</pre>

这五个偏误，火鸡全犯了。人类也全犯。

## 图三：从火鸡到系统设计者

<pre class="mermaid">

graph LR
    A[🔥 火鸡] -->|认清系统| B[👀 观察者]
    B -->|理解规则| C[🌾 农夫]
    C -->|设计规则| D[🏗️ 系统设计者]

    style A fill:#d32f2f,color:#fff
    style D fill:#2e7d32,color:#fff

</pre>

三步进化：

1. **火鸡→观察者**：不再只看投喂，开始问"谁在喂、为什么喂、什么时候停"
2. **观察者→农夫**：理解规则，参与规则，获得喂养他人的能力
3. **农夫→设计者**：不只遵守系统，开始设计系统

## 一张自测卡

| 你是哪种角色？ | 标志 |
|---------------|------|
| 🔥 火鸡 | 安全感建立在"对方没有杀心"上 |

| 🌾 农夫 | 有自己的生产和输出 |
| 🏗️ 设计者 | 定义自己的游戏规则 |
